# XSBench Benchmark Results - Page Table Replication Comparison

**Date:** January 28, 2026  
**Benchmark:** XSBench v18 (Argonne National Laboratory)  
**Working Set:** ~100 GB (Est. Memory Usage: 100,482 MB)  
**Configuration:** 5M particle histories, 200K gridpoints/nuclide, 71M unionized energy gridpoints

## System Configuration

- **Threads:** 64
- **OpenMP Settings:** OMP_PROC_BIND=SPREAD, OMP_SCHEDULE=STATIC
- **NUMA Nodes:** 2 (inferred from places: {0,32}...{31,63})

## Key Findings

### 1. Page Table Replication Effectiveness
- **Mitosis reduces DTLB load walk cycles by 3.46×** (8.10T → 2.34T) vs Linux baseline
- **Hydra reduces DTLB load walk cycles by 3.38×** (8.10T → 2.40T) vs Linux baseline
- With interleaving, Mitosis achieves **4.22× reduction** (8.10T → 1.92T)

### 2. Memory Interleaving Impact
- Interleaving alone improves Linux performance by **2.20×**
- Combined with page table replication (Mitosis), achieves **2.80×** improvement

### 3. Consistency
- Mitosis with interleaving shows exceptional consistency (σ = 0.01s for runtime)
- Hydra shows more variance, especially without interleaving (σ = 7.07s)

### 4. Best Configuration
- **Mitosis -r -i all** provides the best performance:
  - 68.77s average runtime
  - 2,471,948 lookups/s
  - 2.80× speedup over baseline

## Command Reference

```bash
# Linux baseline
numactl -P /usr/bin/time -v -- ./bench_xsbench_mt -- -p 5000000 -g 200000

# Linux with interleaving
numactl -P -i all /usr/bin/time -v -- ./bench_xsbench_mt -- -p 5000000 -g 200000

# Mitosis with replication
numactl -r all /usr/bin/time -v -- ./bench_xsbench_mt -- -p 5000000 -g 200000

# Mitosis with replication + interleaving
numactl -r all -i all /usr/bin/time -v -- ./bench_xsbench_mt -- -p 5000000 -g 200000
```
